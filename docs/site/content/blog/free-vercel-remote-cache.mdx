<h1 className="text-center">Vercel Remote Cache가 이제 무료입니다</h1>

Turborepo의 가장 중요한 기능 중 하나는 [원격 캐싱](/docs/core-concepts/remote-caching)입니다. 이는 개발자와 CI가 동일한 작업을 두 번 수행하지 않도록 보장하는 분산 캐싱 레이어입니다.

오늘부터 [Vercel Remote Cache](https://vercel.com/docs/monorepos/remote-caching)는 Vercel에 연결된 모든 Turborepo에 대해 무료로 제공되는 설정 불필요 옵션이 되었으며, [커뮤니티에서 만든 오픈소스 원격 캐시](/docs/core-concepts/remote-caching#community-implementations)와 함께 개발 속도를 높이는 무료 옵션으로 제공됩니다.

## 저장소를 Vercel Remote Cache에 연결하기

### Vercel에서

Vercel에서 `turbo`를 사용하는 명령은 자동으로 Vercel Remote Cache를 사용하도록 구성됩니다. 빌드는 설정 없이도 원격 캐싱의 이점을 누릴 수 있습니다.

### 다른 지속적 통합 제공업체에서

`TURBO_TOKEN` 및 `TURBO_TEAM` 환경 변수를 생성하여 각각의 값으로 Vercel Remote Cache에 인증합니다. 다음에 연결하는 방법을 알아보려면 [CI 제공업체 문서](/docs/guides/ci-vendors)를 참조하십시오:

- [GitHub Actions](/docs/guides/ci-vendors/github-actions)
- [GitLab CI](/docs/guides/ci-vendors/gitlab-ci)
- [CircleCI](/docs/guides/ci-vendors/circleci)
- [GitLab CI](/docs/guides/ci-vendors/gitlab-ci)
- [Travis CI](/docs/guides/ci-vendors/travis-ci)
- [기타 CI 제공업체](/docs/guides/ci-vendors#general-setup)

### 로컬 머신에서

로컬에서 시작하려면 다음을 실행하십시오:

```bash title="Terminal"
npx turbo login
npx turbo link
```
