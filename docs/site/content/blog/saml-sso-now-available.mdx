<h1 className="text-center">SAML SSO를 이제 사용할 수 있습니다</h1>

import { Authors } from '#components/authors';

<Authors authors={['jaredpalmer']} />

[WorkOS](https://workos.com/) 덕분에 GitHub, GitLab, 비밀번호 없는 인증 외에도 Turborepo는 이제 엔터프라이즈 고객을 위해 다음 SAML 제공업체의 Single Sign-on(SSO)을 지원합니다:

- AD FS SAML
- Auth0 SAML
- Azure AD SAML
- CyberArk SAML
- Generic SAML
- G Suite OAuth
- G Suite SAML
- JumpCloud SAML
- Microsoft OAuth
- Okta SAML
- OneLogin SAML
- OpenID Connect
- PingFederate SAML
- PingOne SAML
- Shibboleth
- VMWare SAML

또한 다음을 통해 SCIM(일명 "디렉터리 동기화")을 지원합니다:

- Azure AD SCIM
- BambooHR
- G Suite
- Gusto
- Hibob
- Okta SCIM v1.1
- Okta SCIM v2.0
- Rippling
- SCIM v1.1
- SCIM v2.0
- Workday

귀하의 팀이 SAML SSO 활성화에 관심이 있으시면 [turborepo@vercel.com](mailto:turborepo@vercel.com)으로 문의해 주세요.

## 다음은 무엇인가요?

우리는 보안을 매우 중요하게 생각합니다. SSO가 2021년에 필수적인 것은 분명하지만, 현재 제3자 SOC 2 Type 2 검사도 진행 중입니다. 마지막으로 올 가을에 생체 인식/U2F 액세스 제어 및 감사 로그를 추가하고 있습니다. 업데이트를 기대해 주세요.
