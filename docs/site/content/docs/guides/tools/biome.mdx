[Biome](https://biomejs.dev/)은 JavaScript, TypeScript, JSX, JSON을 위한 빠른 포매터로 CI와 개발자 시간을 절약합니다.

<CreateTurboCallout />

## Turborepo와 함께 Biome 사용하기

Biome은 Turborepo와 함께 사용되는 대부분의 도구와는 달리 **매우 빠른** 예외적인 도구입니다. 이러한 이유로, 각 패키지에 별도의 스크립트를 만드는 대신 [Root Task](/docs/crafting-your-repository/configuring-tasks#registering-root-tasks)를 사용하는 것을 권장합니다.

<Callout type="info" title="캐싱 동작">
  프로젝트 루트에서 Biome을 사용하면 Biome 버전을 업그레이드하거나 설정을 변경할
  때 모든 작업에 대해 캐시 미스가 발생합니다. 이러한 상황에서 적은 설정보다 높은
  캐시 적중률의 장단점을 선호하는 경우, 가이드의 다른 권장 사항처럼 별도의
  스크립트에서 Biome을 사용할 수 있습니다.
</Callout>

### Biome 초기화

먼저, 저장소에서 [Biome 설정을 위한 설치 문서](https://biomejs.dev/guides/getting-started/)를 따르세요. 그런 다음 저장소의 루트에서 Biome을 사용할 스크립트를 만들 수 있습니다:

```json title="./package.json"
{
  "scripts": {
    "format-and-lint": "biome check .",
    "format-and-lint:fix": "biome check . --write"
  }
}
```

### Root Task 생성

실제로 Biome은 저장소의 반복 속도에 병목 현상이 될 가능성이 낮습니다. 이러한 이유로, [Root Task](/docs/crafting-your-repository/configuring-tasks#registering-root-tasks)에서 Biome을 사용하여 저장소에서 관리할 설정을 줄일 수 있습니다.

<Callout type="info">
  저장소에서 패키지의 작업으로 분할하는 것이 Biome이 더 빠를 것이라고 생각한다면
  그렇게 하셔도 됩니다. 사용 사례에 가장 적합한 것을 실험해 보시기 바랍니다.
</Callout>

[Root Task](/docs/crafting-your-repository/configuring-tasks#registering-root-tasks)를 생성하려면 스크립트를 Turborepo에 등록하세요:

```json title="./turbo.json"
{
  "tasks": {
    "//#format-and-lint": {},
    "//#format-and-lint:fix": {
      "cache": false
    }
  }
}
```

이제 `turbo run format-and-lint`와 `turbo run format-and-lint:fix`를 사용하여 이러한 스크립트를 실행할 수 있습니다.
