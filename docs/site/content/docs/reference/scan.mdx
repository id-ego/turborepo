---

하나의 대화형 명령어로 Turborepo에 대한 더 빠른 툴링을 로컬에서 활성화합니다.

```bash title="Terminal"
turbo scan
```

Turborepo에 대한 가장 빠른 설정을 활성화하기 위한 짧은 일련의 단계를 거치게 됩니다. 이러한 최적화에는 다음이 포함됩니다:

- **Git FS Monitor**: `turbo`는 파일 변경 검색을 위해 Git에 의존합니다. `git`이 변경 사항을 알려줄 때까지 기다려야 하므로 [Git의 내장 파일시스템 모니터](https://git-scm.com/docs/git-fsmonitor--daemon)를 사용하여 이러한 알림을 더 빨리 받을 수 있습니다.
- **Turborepo Daemon**: Turborepo의 데몬은 백그라운드에서 저장소를 낙관적으로 이해합니다. 태스크를 실행할 때 `turbo`가 필요로 할 공통 작업을 미리 수행함으로써 `turbo`는 태스크를 더 빠르게 실행합니다.
- **Remote Caching**: Turborepo는 로컬에서 작업을 캐시할 수 있지만 모든 머신에서 캐시를 공유할 수도 있습니다. [Remote Caching](/docs/core-concepts/remote-caching)을 활성화하면 캐싱이 **멀티플레이어**가 되어 귀하, 팀원 및 CI 머신이 동일한 작업을 두 번 수행하지 않도록 보장합니다.
- **`turbo` 버전 확인**: 우리는 항상 `turbo`를 더 좋게 만들기 위해 노력하고 있습니다. Turborepo의 최신 버전을 사용하고 있는지 확인하기 위해 버전을 확인하고 아직 사용하지 않는 경우 `latest`를 설치하도록 알려드립니다.
- **Turborepo LSP 확인**:
  [VSCode Extension Marketplace](https://marketplace.visualstudio.com/items?itemName=Vercel.turbo-vsc)를 방문하여 IDE용 Turborepo LSP 확장 프로그램을 설치하세요.
